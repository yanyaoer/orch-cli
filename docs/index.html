Daemonless multi-agent orchestration

无常驻服务的多 Agent 编排

Run coding agents as reproducible local jobs.

把编码 Agent 变成可复现的本地任务。

orch starts Codex, Claude, or pi in a supervised headless run, captures local state, validates a role-specific result, and mirrors decisions to PRs or MRs only when you ask it to.

orch 用 supervisor 启动 Codex、Claude 或 pi headless worker，落盘本地状态，校验统一结果，并只在你明确执行时把裁决镜像到 PR 或 MR。

Start with the CLI 从 CLI 开始 View architecture 查看架构 Latest release 最新发布 
$ orch run create --mr 123 --role reviewer --agent codex \
  --tag review-a --worktree . --task task.md

Local state first. Provider streams second.

本地状态优先，模型原始流其次。

The controller does not have to remember what happened. orch snapshots the task, records lifecycle events, keeps native provider output, and validates one role-specific result before marking the run done.

主控不需要把上下文记在聊天里。orch 会快照任务、记录生命周期事件、保留 provider 原始输出，并在终态前校验角色专属的结果 JSON。

Daemonless

无 daemon

Each command reads local state, advances one action, and exits. The supervisor lives only for a single run.

每个命令读取本地状态、推进一次动作，然后退出。supervisor 生命周期只覆盖单个 run。

Auditable

可审计

A run directory contains spec, status, normalized events, native logs, and result artifacts.

每个 run 目录都包含 spec、status、标准事件、原始日志和结果产物。

Mirror-safe

镜像安全

PR/MR writes go through an outbox and stay dry-run until `--execute` is passed.

PR/MR 写入先进入 outbox；没有 `--execute` 就只打印计划命令。

Quickstart

快速开始

Install with Bun or a release binary, then dispatch a role-scoped task to Codex, Claude, or pi.

使用 Bun 或 release binary 安装，然后把带角色边界的任务派发给 Codex、Claude 或 pi。 

$ bun install
$ bun test
$ bun run build
$ cp ./dist/orch ~/.local/bin/orch

$ orch run create \
  --mr 123 \
  --role reviewer \
  --agent codex \
  --tag review-a \
  --worktree . \
  --task task.md

$ orch status --mr 123 --json
$ orch result --mr 123 --run review-a-...

$ curl -L https://github.com/yanyaoer/orch-cli/releases/latest/download/orch-darwin-arm64 -o ~/.local/bin/orch
$ chmod +x ~/.local/bin/orch

$ XDG_STATE_HOME=/tmp/orch-state orch run create --mr demo --role reviewer --agent codex --tag review-a --worktree . --task task.md
