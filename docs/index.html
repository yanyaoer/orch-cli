Daemonless multi-agent orchestration

无常驻服务的多 Agent 编排

Run coding agents as reproducible local jobs.

把编码 Agent 变成可复现的本地任务。

orch dispatches Codex, Claude, pi, or agy through a signed local mail bus, claims work atomically, validates role-specific results, and routes review/verify only after an implementer result is submitted.

orch 通过签名本地 mail bus 派发 Codex、Claude、pi 或 agy，原子 claim 任务，校验角色结果，并在 implementer 提交结果后再派发 review/verify。

Start with the CLI 从 CLI 开始 View architecture 查看架构 Latest release 最新发布 
$ orch mail submit --thread th_123 --mr 123 --workspace orch-cli --task task.md
$ RAW=$(orch mail deliver-local --thread th_123 | bun -e 'const p=JSON.parse(await Bun.stdin.text()); console.log(p.delivered[0]?.to ?? "")')
$ orch mail import --thread th_123 --file "$RAW"
$ orch mail route --thread th_123
$ orch mail claim --thread th_123 --agent codex-implementer

Signed local mail is the bus.

签名本地邮件就是消息总线。

The controller does not need a daemon or external queue. orch stores signed task/result events in Maildir-backed local state, then routes reviewer and verifier work from the implementer's submitted result.

主控不需要 daemon 或外部队列。orch 把签名 task/result 事件写入 Maildir 本地状态，并基于 implementer 提交的结果派发 reviewer 和 verifier。

Daemonless

无 daemon

Each command reads local state, advances one action, and exits. The supervisor lives only for a single run.

每个命令读取本地状态、推进一次动作，然后退出。supervisor 生命周期只覆盖单个 run。

Mail-bus auditable

Mail bus 可审计

A thread contains signed .eml files, verified mail-events.jsonl, Maildir messages, and per-event claim records.

每个 thread 都包含签名 .eml、已验证的 mail-events.jsonl、Maildir 消息和逐事件 claim 记录。

Result-driven review

基于结果的 review

Reviewer and verifier tasks are created from result.submitted, so cross-review audits actual implementer output.

reviewer 和 verifier 任务由 result.submitted 触发，因此交叉 review 审的是真实实现产物。

Quickstart

快速开始

Install with Bun or a release binary, configure default mail agents, then dispatch a task through the local mail bus.

使用 Bun 或 release binary 安装，配置默认 mail agents，然后通过本地 mail bus 派发任务。 

$ bun install
$ bun test
$ bun run install:local

$ orch mail agent defaults
$ orch workspace add --id orch-cli --path .
$ orch mail submit --thread th_123 --mr 123 --workspace orch-cli --task task.md
$ orch mail deliver-local --thread th_123
$ orch mail import --thread th_123 --file 
$ orch mail route --thread th_123
$ orch mail claim --thread th_123 --agent codex-implementer

$ orch mail reply result --thread th_123 --run impl-a-... --from-agent codex-implementer
$ orch mail deliver-local --thread th_123
$ orch mail import --thread th_123 --file 
$ orch mail route --thread th_123
$ orch mail claim --thread th_123 --agent claude-reviewer
$ orch mail claim --thread th_123 --agent pi-verifier

Boundaries

边界

  • No long-running daemon yet.
  • The shipped message bus is local Maildir state, not a multi-machine database.
  • Drivers target Codex, Claude, pi, and agy. Permissions match the role: reviewer runs read-only/sandboxed; agy (Gemini 3.1 Pro) is reviewer-only.
  • 当前没有常驻 daemon。
  • 当前消息总线是本地 Maildir 状态，不是多机数据库。
  • 当前 driver 覆盖 Codex、Claude、pi 和 agy；权限随角色匹配:reviewer 以只读/沙箱方式运行,agy(Gemini 3.1 Pro)仅限 reviewer。
  • chatgpt-bridge 提供只读 Cloudflare Worker MCP 桥接。
