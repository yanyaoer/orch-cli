坑 1 · agent 原地跑真仓库

现状无 worktree 副本（--worktree 默认 cwd）。VM 内以 只读快照 + RW overlay 挂 worktree，退出只回吐 diff，overlay 随 VM 销毁消失。

坑 2 · 运行态要桥回宿主

driver 退出后要读 native.jsonl / result.json / last_message.txt（在 ~/.local/state/orch）。故 driver 留宿主，用 matchlock VFS 把这几个产物桥出 guest。

坑 3 · guest 是 Linux

claude/codex/pi + Bun 工具链、fish wrapper、buildWorkerEnv 都要在 Linux guest 内重放。先花 15 分钟证伪能否 headless 跑通，再投集成。

源码内嵌本页，渲染默认走 plantuml.com（会把图文外发）。介意就把上方 server 改成自建 plantuml/plantuml-server（同样支持 ~h 十六进制编码）。 参考：jingkaihe/matchlock。
